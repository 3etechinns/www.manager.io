Linode

If you don't have an account. Sign up for new Linode account and go through verification process.

You can use promo code DOCS10 for $10 credit on a new account which is enough to run Manager on Linode servers for 2 months.

Creating a server

Once you are logged in, go to Linode tab to select your server size. You can select the cheapest $5 per month option.

Then select location where you want your server to be launched.

Finally, click Add This Linode! button to start the server.

In control panel, you will see your new server being added with status Being Created

By default backups are disabled. We strongly encourage you to enable backups.

Also, note IP address of your server. You will need it later when connecting to your server.

Installing Manager

Click on Dashboard under Options link for your newly created server.

Then click on Rebuild tab.

Under Rebuild tab, click on Deploying using StackScripts

Under Community StackScripts, you will need to find the one that will install Manager. In order to do that, type mngr into Keywords field and then click Search button.

Find the one which says mngr / Manager - Server Edition and click on it.

On the next screen, don't change any options. Just enter new password to Root Password field and click Rebuild button.

Once tasks in Host Job Queue are completed, click Boot button.

Your server will be powered on but it will take another 5-10 minutes for Manager to be installed on it. Be patient.

Connecting to the Server

After you wait for 5-10 minutes, Manager should be installed on your new server.

Open your web-browser and enter xx.xx.xx.xx:8080. where xx.xx.xx.xx is IP address of your server. So for example, if your IP address is 172.104.51.67 then you should enter to your web-browser 172.104.51.67:8080

You should see login screen to Manager Server.

If this is what you see. Congratulations, you are up & running. Just click Login button as default password is empty.

It is highly recommended that you don't connect to your server directly via IP address for security reasons. You should have a domain name pointing to IP address of your server.

For example, if you own domain name mycompany.com, you should create sub-domain accounting.mycompany.com or similar and have it pointed to IP address of your server. If you are not sure how to do this, reach out to your domain registrar or local IT person.

Your server will automatically request free SSL certificate for the domain you choose to use and your connection to your server will be automatically encrypted using HTTPS protocol.
